# 쥬얼리아 (Jewelria) 웹게임 기획서

> **프로젝트명**: 쥬얼리아 (Jewelria)  
> **장르**: 캐주얼 매치-3 퍼즐  
> **플랫폼**: 웹(PWA) — PC / 모바일 브라우저  
> **개발 기술**: Vanilla JavaScript (ES6+), CSS3, HTML5  
> **버전**: 0.1 (기획 초안)  
> **작성일**: 2026-05-30

---

## 1. 개요

### 1.1 게임 소개
같은 색상의 보석을 3개 이상 가로/세로로 정렬하여 제거하는 매치-3 퍼즐 게임이다. 반짝이는 보석을 맞춰 사라뜨리는 직관적인 조작과 짧은 플레이 타임으로 누구나 즐길 수 있는 캐주얼 게임을 목표로 한다.

### 1.2 핵심 재미 (Core Loop)
```text
보석 스왑 → 매치 발생 → 보석 제거 & 점수 획득
   → 빈 공간 채우기 → 연쇄(콤보) 발생 → 목표 달성 → 다음 스테이지
```

### 1.3 타겟 유저
- 가벼운 퍼즐을 즐기는 캐주얼 게이머
- 짧은 시간(1~3분) 플레이를 선호하는 모바일 유저
- 전 연령층

---

## 2. 게임 시스템

### 2.1 보드 (Game Board)
- **크기**: 8 x 8 그리드 (기본값, 스테이지별 조정 가능)
- **셀(Cell)**: 각 칸에는 하나의 보석이 배치됨
- **초기 배치 규칙**: 시작 시 자동 매치가 없도록 생성

### 2.2 보석 종류 (Gem Types)
| 보석 | 색상 | 비고 |
|------|------|------|
| 루비 | Red | 기본 |
| 사파이어 | Blue | 기본 |
| 에메랄드 | Green | 기본 |
| 토파즈 | Yellow | 기본 |
| 아메시스트 | Purple | 기본 |
| 시트린 | Orange | 기본 |

> 기본 6종. 각 보석은 색상과 함께 컷팅 모양(원형/사각/마름모 등)으로도 구분되어 색약 사용자도 식별 가능하도록 한다.

### 2.3 조작 방식
- **PC**: 마우스 드래그 또는 클릭 후 인접 셀 클릭
- **모바일**: 스와이프(상하좌우)
- 인접한 두 보석만 교환 가능
- 교환 후 매치가 발생하지 않으면 원위치로 복귀

### 2.4 매치 규칙
- **기본 매치**: 같은 색 보석 3개 이상 가로 또는 세로 정렬
- **연쇄(Cascade/Combo)**: 보석 제거 후 떨어진 보석이 새 매치를 형성하면 연쇄 발생, 추가 점수
- **동시 매치**: 한 번의 스왑으로 여러 줄 매치 시 보너스

### 2.5 특수 보석 (Special Gem)
| 특수 보석 | 생성 조건 | 효과 |
|-----------|-----------|------|
| 광선 보석 (Striped) | 4개 일렬 매치 | 가로 또는 세로 한 줄 전체 제거 |
| 폭발 보석 (Wrapped) | L자/T자 매치 | 주변 3x3 영역 2회 폭발 |
| 프리즘 보석 (Color Bomb) | 5개 일렬 매치 | 같은 색 보석 전체 제거 |

> MVP(1차 출시)에서는 광선 보석까지만 구현, 이후 업데이트로 확장.

## 3. 게임 모드 & 목표

### 3.1 스테이지 모드
- 스테이지마다 클리어 목표가 다름
- **이동 횟수 제한** 내에 목표 달성

### 3.2 목표 유형
| 유형 | 설명 |
|------|------|
| 점수 달성 | 제한된 이동 횟수 내 목표 점수 도달 |
| 보석 수집 | 특정 색 보석 N개 제거 |
| 장애물 제거 | 광물/얼음 등 특수 블록 모두 제거 (확장) |

### 3.3 별점 평가
- 클리어 점수에 따라 1~3개의 별 부여
- 별 개수가 다음 스테이지 해금 및 진행도에 반영

---

## 4. 점수 & 진행

### 4.1 점수 체계
- 기본 보석 1개 제거: **10점**
- 4매치: **+20 보너스**
- 5매치 / 특수 보석 발동: **+50 보너스**
- 연쇄(콤보) 배수: 콤보 단계마다 점수 배율 증가 (x1 → x1.5 → x2 ...)

### 4.2 진행 데이터 (저장)
- `localStorage`에 저장 (서버 없이 로컬 진행)
  - 현재 스테이지 / 최고 점수 / 획득 별 개수 / 사운드 설정
- 추후 Cloudflare Workers 백엔드 연동으로 클라우드 저장 확장 가능

## 5. UI / UX 구성

### 5.1 화면 구성
1. **타이틀 / 메인 화면**
   - 게임 시작, 스테이지 선택, 설정 버튼
2. **스테이지 선택 화면**
   - 잠금/해금 상태, 획득 별 표시
3. **인게임 화면**
   - 상단: 목표, 남은 이동 횟수, 현재 점수
   - 중앙: 게임 보드
   - 하단: 일시정지 / 다시하기 버튼
4. **결과 화면**
   - 클리어/실패, 획득 점수, 별점, 다음 / 재도전 버튼

### 5.2 시각 효과
- 보석 제거 시 반짝임 파티클/페이드 애니메이션
- 연쇄 발생 시 콤보 텍스트 팝업
- 특수 보석 발동 시 빛 번짐 강조 효과

### 5.3 사운드
- BGM (메인 / 인게임)
- 효과음: 스왑, 매치, 콤보, 클리어, 실패
- 음소거 토글 (`localStorage` 저장)

## 6. 기술 구조

### 6.1 폴더 구조
```text
jewelria/
├── index.html
├── index-en.html
├── index-ja.html
├── manifest.json
├── service-worker.js
├── version.json
└── assets/
    ├── css/
    │   └── style.css
    ├── js/
    │   ├── main.js
    │   ├── board.js
    │   ├── gem.js
    │   ├── input.js
    │   ├── score.js
    │   ├── stage.js
    │   ├── ui.js
    │   ├── audio.js
    │   ├── storage.js
    │   ├── ga.js
    │   └── browser-check.js
    ├── images/
    │   └── gems/
    └── sounds/
```

### 6.2 핵심 알고리즘
- **매치 탐지**: 보드 전체를 스캔하여 가로/세로 3개 이상 동일 색 그룹 탐색
- **중력 처리**: 보석 제거 후 위쪽 보석을 아래로 낙하시키고 빈 칸을 새 보석으로 채움
- **연쇄 처리**: 낙하 완료 후 재스캔하여 새 매치가 없을 때까지 반복
- **렌더링**: MVP는 CSS Grid + DOM 기반

### 6.3 PWA
- `manifest.json`으로 홈 화면 추가 지원
- `service-worker.js`로 정적 자산 캐싱 및 오프라인 플레이
- 다국어 매니페스트는 추후 확장

### 6.4 다국어 지원
- 언어별 HTML 파일 분리 (`index.html`, `index-en.html`, `index-ja.html`)
- `hreflang` / `canonical` 메타 태그로 SEO 대응
- UI 문자열은 JS 객체로 분리 관리

## 9. 참고 사항
- 보석 그래픽은 저작권 문제가 없는 자체 제작 또는 생성형 에셋만 사용
- 특정 유명 매치-3 게임의 이름/로고/UI 디자인을 모방하지 않음
- 게임명 "쥬얼리아(Jewelria)"는 조어(造語)로 고유성이 높으나, 정식 서비스 전 [키프리스(KIPRIS)](http://www.kipris.or.kr) 상표 검색으로 중복 여부를 확인 권장
