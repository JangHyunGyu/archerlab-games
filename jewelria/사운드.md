# 쥬얼리아 사운드 가이드

> 목적: 쥬얼리아의 사운드 구조와 디자인 의도를 설명하는 문서  
> 적용 위치: 효과음(SFX)은 코드 합성, BGM은 `jewelria/assets/sounds/`

## 1. 사운드 아키텍처

쥬얼리아의 사운드는 두 계층으로 나뉩니다.

- **효과음(SFX)**: 100% WebAudio 실시간 합성. 바이너리 오디오 파일을 쓰지 않음.
  - 위치: `assets/js/audio.js`의 `AudioManager` 클래스
  - 장점: 0바이트(다운로드/캐시 부담 없음), 재생마다 미세 변주로 반복 피로 감소, 라이선스 안전, 코드로 즉시 튜닝
- **BGM**: mp3 루프 파일 사용.
  - `assets/sounds/bgm_main_loop.mp3` (메인 화면)
  - `assets/sounds/bgm_game_loop.mp3` (인게임)

## 2. 전체 사운드 방향

- 톤: 고급 보석, 유리, 크리스털, 금속성 반짝임, 깊은 임팩트
- 느낌: 보석이 깨지고 흩어지는 시원한 타격감, 콤보가 커질수록 더 크고 화려한 상승감
- 피해야 할 것: 캔디류 게임을 직접 연상시키는 말랑한 사탕 소리, 캐릭터 보이스, 긴 잔향, 과도한 폭발음
- 기준: 모바일 스피커에서도 잘 들리는 선명한 고역 + 저역 임팩트, 이어폰에서는 날카롭지 않게

## 3. 효과음(합성) 이벤트 매핑

`audio.play(name, intensity, detail)`로 호출되는 이벤트별 합성 사운드입니다.

| 이벤트 | 사용 상황 | 합성 구성 |
|---|---|---|
| `swap` | 보석 교환 | 짧은 고역 노이즈 + 글라이드 트라이앵글 |
| `invalid` | 매치 실패 후 복귀 | 낮은 톱니/삼각 톤 + 둔탁한 로우패스 노이즈 |
| `match` | 매치 성사 | 유리 파열 노이즈 + 비정수배 배음 종소리 + 저역 임팩트 + 반짝임 꼬리 |
| `combo` | 콤보 | 파열 + 필터 스윕 + 상승 아르페지오 + 반짝임 (레벨에 따라 음정/길이 증가) |
| `special` | 특수 보석 발동 | 프리즘: 강한 파열 + 양방향 스윕 + 6음 상승 + 긴 반짝임 블룸 |
| `cascade` | 연쇄 낙하 | 하강 글라이드 + 보석 착지 입자음 + 약한 반짝임 |
| `clear` | 스테이지 성공 | 메이저 상승 아르페지오 + 반짝임 블룸 팡파르 |
| `fail` | 스테이지 실패 | 하강 3음 스팅어(우울하지 않게) |
| `button` | 버튼 클릭 | 골드 클릭 톤 + 짧은 고역 노이즈 |

`detail` 객체로 전달되는 강도 정보:
- `match`/`combo`/`special`: `{ lineCount, longest, removedCount, specialActivations }`
- `cascade`: `{ fallCount }`

라인 수/최장 매치/낙하량이 클수록 음정, 음량, 반짝임 밀도가 함께 커집니다.

## 4. 합성 빌딩 블록 (audio.js)

| 메서드 | 역할 |
|---|---|
| `gemBell` | `GEM_PARTIALS` 비정수배 배음으로 만든 크리스털 종소리 |
| `glassShatter` / `glassNoise` | 필터된 노이즈 버스트(유리 깨짐 트랜지언트) |
| `subImpact` | 피치 드롭 저역 바디 임팩트 |
| `sparkleTail` | 랜덤 사인 그레인 고역 반짝임 꼬리 |
| `sweep` | 밴드패스 필터 스윕(에너지 상승감) |
| `glide` / `partial` | 글라이드 톤 / 단일 배음 톤 |
| shimmer 센드 버스 | 피드백 딜레이 + 하이패스로 크리스털 잔향 부여 |

신호 체인: 각 보이스 → `StereoPanner`(스테레오 폭) → 마스터 게인(0.4) → `DynamicsCompressor`(리미터) → 출력. 일부 보이스는 shimmer 센드로 잔향을 추가합니다.

## 5. BGM (mp3 루프)

| 파일명 | 사용 상황 | 권장 길이 | 메모 |
|---|---:|---:|---|
| `bgm_main_loop.mp3` | 메인 화면 BGM | 30~60초 루프 | 고급 보석 상점, 판타지 라운지 느낌 |
| `bgm_game_loop.mp3` | 인게임 BGM | 45~90초 루프 | 집중을 방해하지 않는 리듬, 밝고 반복 피로감 낮게 |

- `startAmbient('main' | 'game')`로 전환, `stopAmbient()`로 페이드 아웃.
- BGM 음량은 `playMusic`에서 메인 0.088 / 인게임 0.105로 조정됩니다.

## 6. 외부 SFX 파일 사양 (직접 제작/구매분 반입)

합성 대신 **외부에서 가져온 실음원**으로 교체하려면 아래 사양에 맞춰 파일을 넣어주세요.
파일이 존재하면 코드가 자동으로 합성 대신 해당 음원을 재생하고, 없는 이벤트는 기존 합성으로 폴백하도록 연동하겠습니다(샘플 우선 + 합성 폴백 하이브리드).

### 6-1. 배치 위치
```
assets/sounds/sfx/
```
- BGM(`assets/sounds/*.mp3`)과 폴더를 분리해 관리합니다.

### 6-2. 공통 기술 사양

| 항목 | 권장값 | 비고 |
|---|---|---|
| 포맷 | `.mp3` (필수) | 전 브라우저/iOS 호환. 원하면 `.ogg`도 같이 두면 코드가 우선 사용 |
| 샘플레이트 | 44.1 kHz | 원본 48 kHz면 그대로도 무방 |
| 채널 | 스테레오 | 모노여도 동작(공간감은 코드 패닝/리버브가 보강) |
| 비트레이트 | 192 kbps 이상 | SFX는 용량 작으니 256 kbps 권장 |
| 라우드니스 | 약 -14 LUFS, 트루피크 -1 dBTP | 이벤트 간 음량 균형이 핵심. 클리핑 금지 |
| 앞뒤 무음 | **반드시 트림** | 앞 무음 = 반응 지연. 끝은 5~10ms 페이드로 클릭음 제거 |
| 길이 | 이벤트별 표 참고 | 짧을수록 타격감 ↑ |

### 6-3. 파일명 ↔ 이벤트 매핑 (필수 9종)

| 파일명 | 이벤트 | 권장 길이 | 음향 방향 |
|---|---|---:|---|
| `swap.mp3` | 보석 교환 | 80~160 ms | 짧고 가벼운 "스윽/톡" |
| `invalid.mp3` | 매치 실패 복귀 | 150~300 ms | 둔탁한 "음-" 거절음(불쾌하지 않게) |
| `match.mp3` | 일반 매치 | 250~500 ms | 유리/크리스털 깨짐 + 영롱한 잔향 |
| `combo.mp3` | 콤보 | 400~800 ms | 깨짐 + 상승감, 밝고 화려하게 |
| `special.mp3` | 특수 보석 발동 | 600~1200 ms | 프리즘 폭발 + 긴 반짝임 블룸 |
| `cascade.mp3` | 연쇄 낙하 | 200~500 ms | 보석이 쏟아져 착지하는 짤랑임 |
| `clear.mp3` | 스테이지 성공 | 1.0~2.0 s | 보석 팡파르, 승리감 |
| `fail.mp3` | 스테이지 실패 | 0.8~1.5 s | 하강 스팅어(너무 우울하지 않게) |
| `button.mp3` | 버튼 클릭 | 40~120 ms | 고급스러운 "딸깍" |

### 6-4. 변주/강도 변형 (선택 — 품질·반복 피로 개선에 강력 추천)

같은 이벤트라도 매번 똑같으면 싸구려로 들립니다. 아래처럼 **번호 변주**를 두면 코드가 매번 랜덤으로 골라 재생합니다.

```
match_1.mp3, match_2.mp3, match_3.mp3      # 랜덤 변주
combo_1.mp3, combo_2.mp3
button_1.mp3, button_2.mp3
```

매치는 규모별 **강도 티어**도 둘 수 있습니다(없으면 `match.mp3` 하나로 모든 규모 처리, 코드가 음량/피치로 보강):

```
match_small_1.mp3, match_small_2.mp3       # 3개 매치
match_big_1.mp3,   match_big_2.mp3         # 라인 2개 이상 / 4매치
match_huge.mp3                             # 5매치 / 대형 연쇄
```

- 규칙: 기본 이름(`match.mp3`)이 있으면 그것을, `match_숫자` 변주가 있으면 그중 랜덤, `match_티어_숫자`가 있으면 강도에 맞춰 우선 선택. 우선순위는 **티어 > 변주 > 기본 > 합성 폴백**.

### 6-5. 음향 디자인 가이드(섹션 2 톤 유지)

- 톤: 고급 보석·유리·크리스털·금속성 반짝임. **사탕/말랑 사운드, 캐릭터 보이스 금지.**
- 저역 임팩트는 짧게, 잔향(리버브)은 음원 자체에 길게 넣지 말 것 — 공간계는 코드가 입힙니다(이중 리버브 방지).
- 모바일 스피커에서 선명한 고역 + 이어폰에서 날카롭지 않은 균형.

### 6-6. 반입 후 처리(내가 할 일)

파일을 넣고 알려주시면:
1. `AudioManager`에 **샘플 우선 로더**를 추가 — 시작 시 `decodeAudioData`로 프리로드, 원샷 재생을 마스터 체인(에어/컴프/리미터/리버브)에 연결.
2. 변주/티어 자동 선택 + 없는 이벤트는 기존 합성으로 폴백.
3. `service-worker.js` 캐시 목록에 `assets/sounds/sfx/*` 추가, `version.json` 버전 갱신(PWA 캐시 무효화).

## 8. 유지보수 시 주의

- SFX 사운드를 조정하려면 `assets/js/audio.js`의 해당 이벤트 메서드(`playMatch` 등)와 빌딩 블록 파라미터만 수정하면 됩니다. 별도 에셋 작업 불필요.
- BGM 파일을 교체하면 `service-worker.js`의 캐시 목록과 `version.json` 버전을 함께 갱신해야 PWA 캐시 문제가 줄어듭니다.
- 보석 낙하/착지 사운드는 한 번의 낙하 묶음마다 1~2회만 재생하는 방식이 좋습니다.
