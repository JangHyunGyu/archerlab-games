# 쥬얼리아 사운드 제작 가이드

> 목적: 현재 게임 코드에 바로 넣을 수 있는 상업용 캐주얼 매치-3 퍼즐 사운드 에셋 제작 지시서  
> 적용 위치: `jewelria/assets/sounds/`

## 1. 전체 사운드 방향

- 톤: 고급 보석, 유리, 크리스털, 금속성 반짝임, 깊은 임팩트
- 느낌: 보석이 깨지고 흩어지는 시원한 타격감, 콤보가 커질수록 더 크고 화려한 상승감
- 피해야 할 것: 캔디류 게임을 직접 연상시키는 말랑한 사탕 소리, 캐릭터 보이스, 긴 잔향, 과도한 폭발음
- 기준: 모바일 스피커에서도 잘 들리는 선명한 고역 + 저역 임팩트, 이어폰에서는 날카롭지 않게

## 2. 파일 규격

- 효과음 포맷: `wav`
- 샘플레이트: 44.1kHz 또는 48kHz
- 비트뎁스: 16-bit 이상
- 채널: 짧은 UI 효과음은 mono 또는 narrow stereo, 콤보/클리어는 stereo 권장
- 피크: -1.0 dB 이하
- 무음: 시작 무음 10ms 이하, 끝 무음 80ms 이하
- 루프 BGM을 추가할 경우: `mp3` + `ogg` 동시 준비 권장

## 3. 현재 코드에서 바로 교체 가능한 파일

아래 8개 파일명은 현재 `assets/js/audio.js`에서 이미 로드합니다. 같은 이름으로 덮어쓰면 코드 수정 없이 바로 적용됩니다.

| 파일명 | 사용 상황 | 권장 길이 | 제작 방향 |
|---|---:|---:|---|
| `clear_single.wav` | 일반 3매치 | 0.35~0.55초 | 작은 보석 3개가 산산조각 나는 맑은 파열음. 짧고 선명한 크리스털 클릭, 작은 반짝임 꼬리 |
| `clear_double.wav` | 4매치 또는 동시 매치 | 0.50~0.75초 | 더 넓게 깨지는 크리스털 샤터, 중간 저역 임팩트, 반짝임 입자가 좌우로 퍼지는 느낌 |
| `clear_quad.wav` | 5매치급, 특수 효과, 스테이지 클리어 보강 | 0.75~1.05초 | 큰 보석 덩어리가 깨지며 빛나는 파편이 터지는 프리미엄 효과. 저역 펀치 + 고역 글리터 |
| `combo_hit.wav` | 콤보 발생 | 0.40~0.65초 | 콤보 텍스트와 동시에 꽂히는 강한 히트. 금속성 차임과 크리스털 임팩트 |
| `combo_escalate.wav` | 3콤보 이상 상승감 | 0.50~0.80초 | 음정이 위로 올라가는 반짝이는 상승 효과. 다음 콤보를 기대하게 만드는 짧은 리프트 |
| `impact_heavy.wav` | 큰 매치, 특수 보석, 화면 흔들림 보강 | 0.25~0.45초 | 낮고 묵직한 둔탁함 + 유리 파편의 짧은 찰랑임. 모바일에서 진동처럼 느껴지는 저역 |
| `sparkle.wav` | 매치 후 잔광, 클리어 반짝임 | 0.35~0.65초 | 밝고 고급스러운 보석 반짝임. 너무 얇지 않게 여러 겹의 글리터 |
| `whoosh.wav` | 특수 효과, 콤보 스윕 | 0.45~0.75초 | 빛줄기가 지나가는 빠른 스윕. 끝에 작은 크리스털 입자감 |

## 5. 추가 제작 추천 파일

아래 파일들은 아직 코드에 연결되어 있지 않습니다. 가져오면 `assets/js/audio.js`에 추가 연결해서 품질을 더 끌어올릴 수 있습니다.

| 파일명 | 사용 상황 | 권장 길이 | 메모 |
|---|---:|---:|---|
| `swap_slide.wav` | 보석 교환 | 0.15~0.25초 | 현재는 WebAudio 합성. 실제 에셋으로 바꾸면 조작감이 좋아짐 |
| `invalid_swap.wav` | 매치 실패 후 복귀 | 0.20~0.35초 | 낮고 짧은 되돌림 소리, 불쾌하지 않게 |
| `ui_button.wav` | 버튼 클릭 | 0.10~0.20초 | 골드 버튼 + 작은 보석 클릭 |
| `gem_drop_soft.wav` | 새 보석 낙하 시작 | 0.18~0.30초 | 가벼운 하강 스윽 소리. 너무 자주 재생되므로 작게 |
| `gem_land_sparkle.wav` | 보석 착지 | 0.15~0.28초 | 보석이 칸에 닿는 작은 딸깍 + 반짝임 |
| `cascade_rush.wav` | 연쇄 낙하 전체 | 0.35~0.55초 | 보석들이 한꺼번에 내려오는 광택 있는 러시 |
| `stage_clear_fanfare.wav` | 스테이지 성공 | 1.2~2.0초 | 짧은 승리 팡파르. 보석 반짝임과 밝은 코드 |
| `stage_fail_stinger.wav` | 스테이지 실패 | 0.8~1.4초 | 너무 우울하지 않은 실패 스팅어 |
| `bgm_main_loop.mp3` | 메인 화면 BGM | 30~60초 루프 | 고급 보석 상점, 판타지 라운지 느낌 |
| `bgm_game_loop.mp3` | 인게임 BGM | 45~90초 루프 | 집중을 방해하지 않는 리듬, 밝고 반복 피로감 낮게 |

## 7. 적용 메모

- 기존 8개 파일은 같은 이름으로 `jewelria/assets/sounds/`에 넣으면 바로 사용됩니다.
- 새 파일을 추가하는 경우 `assets/js/audio.js`의 `SOUND_ASSETS`에 경로를 추가하고, 필요한 재생 위치에서 `playBuffer()`로 호출해야 합니다.
- 파일을 추가하거나 교체한 뒤 `service-worker.js`의 캐시 목록과 `version.json` 버전을 함께 갱신해야 PWA 캐시 문제가 줄어듭니다.
- 보석 낙하/착지 사운드는 너무 많은 개별 재생을 피하고, 한 번의 낙하 묶음마다 1~2회만 재생하는 방식이 좋습니다.
