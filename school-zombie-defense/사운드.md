# 스쿨 언데드 디펜스 사운드 정리

> 목적: 다른 ArcherLab 게임에서 재사용한 실제 샘플과, 아직 새로 제작해야 하는 효과음을 구분한다.  
> 현재 상태: `game.js`의 `playSfx(name, intensity)`는 샘플을 우선 재생하고, 샘플이 없거나 아직 로드되지 않았으면 기존 WebAudio 합성음으로 폴백한다.

## 1. 적용 방식

- 샘플 위치: `school-zombie-defense/assets/sounds/sfx/`
- 코드 매핑: `school-zombie-defense/js/game.js`의 `SFX_ASSETS`
- BGM 위치: `school-zombie-defense/assets/sounds/bgm/`
- 로딩 방식: 첫 오디오 unlock 이후 `fetch` + `decodeAudioData`로 비차단 프리로드
- 폴백 방식: 샘플 버퍼가 없으면 기존 `playTone` / `playNoise` 합성 사운드 재생
- BGM 재생: 메뉴/상점/게임오버는 `menu_loop.mp3`, 출격 후 인게임은 `game_loop.mp3`

## 5. 아직 새로 필요한 샘플

현재 기본 사운드는 연결됐고, 아래 항목은 품질을 더 올리고 싶을 때 새로 제작하면 좋습니다.

| 이벤트 | 필요한 파일 | 권장 길이 | 제작 방향 |
|---|---|---:|---|
| `pistol_1~3` | `assets/sounds/sfx/pistol_1.mp3` 등 | 45~110 ms | 권총 발사 변형. 반복 피로도 감소 |
| `rifle_1~3` | `assets/sounds/sfx/rifle_1.mp3` 등 | 35~95 ms | 소총 버스트 변형. 연사 시 겹침 완화 |
| `hit_1~4` | `assets/sounds/sfx/hit_1.mp3` 등 | 60~140 ms | 피격 변형. 같은 살점음 반복 방지 |
| `death_elite` | `assets/sounds/sfx/death_elite.mp3` | 350~800 ms | 엘리트 처치 전용 무게감 |
| `explosion_large` | `assets/sounds/sfx/explosion_large.mp3` | 600~1200 ms | 대형 폭발/특수 폭발 전용 |

## 6. 권장 제작 사양

| 항목 | 권장값 | 메모 |
|---|---:|---|
| 포맷 | mp3 우선, wav 허용 | 현재 재사용 파일은 mp3/wav 혼합. 브라우저 디코딩은 둘 다 가능 |
| 샘플레이트 | 44.1 kHz | 원본 48 kHz도 가능 |
| 비트레이트 | mp3 192 kbps 이상 | SFX는 256 kbps 권장 |
| 라우드니스 | 약 -14 LUFS, TP -1 dB | 여러 SFX가 겹쳐도 찢어지지 않게 |
| 앞 무음 | 0~8 ms | 입력 반응이 늦어 보이지 않게 |
| 뒤 무음 | 20 ms 이하 | 반복 SFX 꼬리가 겹치지 않게 |

## 7. 변형 파일 규칙

반복 빈도가 높은 무기/피격/처치음은 한 파일만 쓰면 금방 질립니다. 새로 제작할 때는 아래처럼 변형을 함께 준비하는 편이 좋습니다.

```text
pistol_1.mp3, pistol_2.mp3, pistol_3.mp3
rifle_1.mp3, rifle_2.mp3, rifle_3.mp3
hit_1.mp3, hit_2.mp3, hit_3.mp3
death_1.mp3, death_2.mp3, death_3.mp3
```

향후 선택 우선순위:

```text
이벤트_상황_번호.mp3 > 이벤트_번호.mp3 > 이벤트.mp3 > 기존 합성 폴백
```

예시:

```text
death_elite_1.mp3
death_normal_1.mp3
explosion_large_1.mp3
hit_crit_flesh_1.mp3
```

## 8. 다음 개선 순서

1. `button`, `denied`, `recruit`, `wave_clear`, `game_over`, `coin`, `pause`를 실제 코드 이벤트로 더 세밀하게 분리
2. 반복 사운드 변형 파일 선택 로직 추가
3. 총기/피격/처치음 길이와 라우드니스 재조정
4. 필요하면 샘플 전체를 mp3 44.1kHz / 192~256kbps로 통일
