# 스쿨 언데드 디펜스 사운드 정리

> 목적: 다른 ArcherLab 게임에서 재사용한 실제 샘플과, 아직 새로 제작해야 하는 효과음을 구분한다.  
> 현재 상태: `game.js`의 `playSfx(name, intensity)`는 샘플을 우선 재생하고, 샘플이 없거나 아직 로드되지 않았으면 기존 WebAudio 합성음으로 폴백한다.

## 1. 적용 방식

- 샘플 위치: `school-zombie-defense/assets/sounds/sfx/`
- 코드 매핑: `school-zombie-defense/js/game.js`의 `SFX_ASSETS`
- 로딩 방식: 첫 오디오 unlock 이후 `fetch` + `decodeAudioData`로 비차단 프리로드
- 폴백 방식: 샘플 버퍼가 없으면 기존 `playTone` / `playNoise` 합성 사운드 재생
- 총기 3종(`pistol`, `rifle`, `sniper`)은 다른 게임에 어울리는 샘플이 없어 현재 합성 폴백 유지

## 2. 바로 적용된 샘플

현재 게임 코드에서 실제로 호출되는 이벤트 중 샘플을 연결한 항목입니다.

| 이벤트 | 대상 파일 | 출처 | 판단 |
|---|---|---|---|
| `start` | `assets/sounds/sfx/start.wav` | `solo-leveling/sounds/select.wav` | 출격 시작용 짧은 확인음으로 적합 |
| `skill` | `assets/sounds/sfx/skill.wav` | `solo-leveling/sounds/levelup.wav` | 스킬 선택/상점 구매 성공에 맞는 보상음 |
| `core` | `assets/sounds/sfx/core.wav` | `solo-leveling/sounds/warning.wav` | 코어 피격, 코인 부족 경고음으로 사용 |
| `hit` | `assets/sounds/sfx/hit.wav` | `solo-leveling/sounds/acid_hit.wav` | 좀비 피격에 쓸 수 있는 짧은 산성/살점 계열 타격음 |
| `crit` | `assets/sounds/sfx/crit.wav` | `solo-leveling/sounds/crit_hit.wav`에서 0.48초 캡 | 원본이 길어 치명타용으로 짧게 가공 |
| `death` | `assets/sounds/sfx/death.mp3` | `solo-leveling/sounds/kill.mp3`에서 0.72초 캡 | 처치 반복에 맞게 원본을 짧게 가공 |
| `explosion` | `assets/sounds/sfx/explosion.wav` | `solo-leveling/sounds/ground_slam.wav` | 로켓 폭발/폭발형 좀비 사망용 저역 임팩트 |
| `rocket` | `assets/sounds/sfx/rocket.wav` | `solo-leveling/sounds/flame_bolt_sfx.wav` | 로켓 추진음 대체 샘플로 임시 사용 |

## 4. 아직 새로 필요한 샘플

다른 게임 파일 중에서는 좀비 디펜스 분위기와 무기감에 충분히 맞는 샘플이 없어서 남겨둔 항목입니다.

| 이벤트 | 필요한 파일 | 권장 길이 | 제작 방향 |
|---|---|---:|---|
| `pistol` | `assets/sounds/sfx/pistol.mp3` | 45~110 ms | 짧고 건조한 핸드건 샷. 반복 발사에 피로하지 않게 |
| `rifle` | `assets/sounds/sfx/rifle.mp3` | 35~95 ms | 빠르고 날카로운 소총 샷. 버스트 연사에서 뭉개지지 않게 |
| `sniper` | `assets/sounds/sfx/sniper.mp3` | 180~420 ms | 깊은 단발감, 짧은 잔향. 너무 길지 않게 |
| 더 나은 `rocket` | `assets/sounds/sfx/rocket.mp3` | 220~500 ms | 실제 로켓 추진음. 현재는 화염탄 샘플을 임시 사용 |
| 더 나은 `arrow` | `assets/sounds/sfx/arrow.mp3` | 80~180 ms | 활시위 + 화살 비행음. 현재는 범용 whoosh를 임시 사용 |
| BGM 메뉴 | `assets/sounds/bgm/menu_loop.mp3` | 45~90 s 루프 | 어두운 학교 복도, 낮은 긴장감 |
| BGM 인게임 | `assets/sounds/bgm/game_loop.mp3` | 60~120 s 루프 | 반복 플레이에 피로하지 않은 전투 리듬 |

## 5. 권장 제작 사양

| 항목 | 권장값 | 메모 |
|---|---:|---|
| 포맷 | mp3 우선, wav 허용 | 현재 재사용 파일은 mp3/wav 혼합. 브라우저 디코딩은 둘 다 가능 |
| 샘플레이트 | 44.1 kHz | 원본 48 kHz도 가능 |
| 비트레이트 | mp3 192 kbps 이상 | SFX는 256 kbps 권장 |
| 라우드니스 | 약 -14 LUFS, TP -1 dB | 여러 SFX가 겹쳐도 찢어지지 않게 |
| 앞 무음 | 0~8 ms | 입력 반응이 늦어 보이지 않게 |
| 뒤 무음 | 20 ms 이하 | 반복 SFX 꼬리가 겹치지 않게 |

## 6. 변형 파일 규칙

반복 빈도가 높은 무기/피격/처치음은 한 파일만 쓰면 금방 질립니다. 새로 제작할 때는 아래처럼 변형을 함께 준비하는 편이 좋습니다.

```text
pistol_1.mp3, pistol_2.mp3, pistol_3.mp3
rifle_1.mp3, rifle_2.mp3, rifle_3.mp3
hit_1.mp3, hit_2.mp3, hit_3.mp3
death_1.mp3, death_2.mp3, death_3.mp3
```

향후 선택 우선순위:

```text
이벤트_상황_번호.mp3 > 이벤트_번호.mp3 > 이벤트.mp3 > 기존 합성 폴백
```

예시:

```text
death_elite_1.mp3
death_normal_1.mp3
explosion_large_1.mp3
hit_crit_flesh_1.mp3
```

## 7. 다음 개선 순서

1. 실제 총기 3종(`pistol`, `rifle`, `sniper`) 신규 샘플 제작/구매
2. `button`, `denied`, `recruit`, `wave_clear`, `game_over`, `coin`, `pause`를 코드 이벤트로 분리
3. 반복 사운드 변형 파일 선택 로직 추가
4. 메뉴/인게임 BGM 추가
5. 필요하면 샘플 전체를 mp3 44.1kHz / 192~256kbps로 통일
