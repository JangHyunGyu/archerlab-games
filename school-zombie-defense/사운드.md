# 스쿨 언데드 디펜스 사운드 샘플 가이드

> 목적: 현재 WebAudio 합성으로 재생되는 효과음을 실제 사운드 샘플로 교체하기 위한 파일별 정의서  
> 적용 대상: `school-zombie-defense/js/game.js`의 `playSfx(name, intensity)` 이벤트

## 1. 사운드 구조

현재 게임은 별도 바이너리 SFX 없이 `game.js` 안에서 `playTone`, `playNoise`로 효과음을 합성합니다. 실제 샘플을 쓰려면 쥬얼리아처럼 **샘플 우선 + 합성 폴백** 구조로 확장하는 것이 좋습니다.

- 샘플 파일이 있으면 `assets/sounds/sfx/*.mp3`를 재생
- 파일이 없거나 로드 실패하면 기존 WebAudio 합성 사운드로 폴백
- 무기 발사와 피격음은 반복 빈도가 높으므로 짧고 선명해야 함
- 폭발, 치명타, 스킬은 리버브와 저역 임팩트를 조금 더 허용

## 2. 배치 경로

```text
school-zombie-defense/
  assets/
    sounds/
      bgm/
        menu_loop.mp3
        game_loop.mp3
      sfx/
        start.mp3
        skill.mp3
        core.mp3
        hit.mp3
        crit.mp3
        death.mp3
        explosion.mp3
        pistol.mp3
        rifle.mp3
        sniper.mp3
        rocket.mp3
        arrow.mp3
```

## 3. 공통 제작 사양

| 항목 | 권장값 | 메모 |
|---|---:|---|
| 포맷 | mp3 | iOS/브라우저 호환 우선. 필요하면 ogg를 보조 포맷으로 추가 |
| 샘플레이트 | 44.1 kHz | 원본이 48 kHz면 그대로도 가능 |
| 비트레이트 | 192 kbps 이상 | SFX는 256 kbps 권장 |
| 채널 | 스테레오 | 무기 발사는 좁게, UI/스킬/폭발은 조금 넓게 |
| 라우드니스 | 약 -14 LUFS, TP -1 dB | 클리핑 금지. 여러 SFX가 겹쳐도 찢어지지 않게 |
| 앞 무음 | 0~8 ms | 반응 지연이 느껴지지 않게 트림 |
| 뒤 무음 | 20 ms 이하 | 짧은 SFX는 꼬리가 과하지 않게 |

## 4. 필수 SFX 파일

현재 코드에서 직접 호출되는 `playSfx` 이벤트 기준입니다.

| 파일명 | 코드 이벤트 | 사용 상황 | 권장 길이 | 사운드 방향 |
|---|---|---|---:|---|
| `start.mp3` | `start` | 출격 시작 | 180~450 ms | 짧은 상승음, 전투 시작 확인감. 과한 팡파르 금지 |
| `skill.mp3` | `skill` | 스킬 선택, 상점 구매 성공 | 350~850 ms | 밝은 전술 보급/강화음. 금속 클릭 + 상승 톤 |
| `core.mp3` | `core` | 방벽/코어 피격, 코인 부족 거절음 | 180~450 ms | 묵직한 경고 타격음. 너무 길거나 불쾌하지 않게 |
| `hit.mp3` | `hit` | 일반 피격 | 60~140 ms | 살점/충돌 느낌의 짧은 둔탁음. 반복 재생에 피로하지 않게 |
| `crit.mp3` | `crit` | 치명타 피격 | 120~260 ms | 일반 피격보다 선명한 금속성 스냅 + 짧은 임팩트 |
| `death.mp3` | `death` | 좀비 처치 | 180~420 ms | 축축한 붕괴/무너짐. 엘리트에도 버틸 만큼 존재감 있게 |
| `explosion.mp3` | `explosion` | 로켓 폭발, 폭발형 좀비 사망 | 450~900 ms | 저역 임팩트 + 짧은 파편음. 모바일 스피커에서도 먹먹하지 않게 |
| `pistol.mp3` | `pistol` | 권총 발사 | 45~110 ms | 짧고 건조한 핸드건 샷. 반복 발사에 적합 |
| `rifle.mp3` | `rifle` | 소총 발사 | 35~95 ms | 빠르고 날카로운 소총 샷. 버스트 연사에서 뭉개지지 않게 |
| `sniper.mp3` | `sniper` | 저격총 발사 | 180~420 ms | 큰 단발감, 깊은 저역과 긴 잔향. 너무 영화 예고편처럼 길지 않게 |
| `rocket.mp3` | `rocket` | 로켓 발사 | 220~500 ms | 발사 추진음 + 짧은 저역. 폭발음과 구분되어야 함 |
| `arrow.mp3` | `arrow` | 활 발사 | 80~180 ms | 활시위 튕김 + 바람 가르는 소리. 총성과 대비되는 가벼움 |

## 5. 권장 확장 SFX

아직 코드 이벤트가 분리되어 있지는 않지만, 실제 샘플 체계로 갈 때 추가하면 좋은 파일입니다.

| 파일명 | 예정 사용 상황 | 권장 길이 | 사운드 방향 |
|---|---|---:|---|
| `button.mp3` | 메뉴/상점 일반 버튼 | 50~120 ms | 작고 단단한 UI 클릭 |
| `purchase.mp3` | 상점 구매 성공 전용 | 300~650 ms | `skill.mp3`보다 짧은 결제/강화 확인음 |
| `denied.mp3` | 코인 부족, 구매 불가 | 150~350 ms | 낮은 톤의 거절음. 실패감은 주되 짜증나지 않게 |
| `recruit.mp3` | 지원 캐릭터 합류 | 600~1100 ms | 캐릭터 합류 팝업용 보상감 |
| `wave_clear.mp3` | 레벨/웨이브 돌파 | 600~1200 ms | 짧은 승리 상승음 |
| `game_over.mp3` | 방어 실패 | 900~1800 ms | 무겁지만 과하게 우울하지 않은 실패음 |
| `coin.mp3` | 코인 획득/정산 | 120~250 ms | 작고 밝은 금속성 보상음 |
| `pause.mp3` | 일시정지/재개 | 80~160 ms | UI 전환 확인음 |

## 6. 변형 파일 규칙

반복 빈도가 높은 무기 발사와 피격음은 한 파일만 쓰면 금방 질립니다. 같은 이벤트에 2~4개 변형을 둘 수 있게 준비하는 편이 좋습니다.

```text
pistol_1.mp3, pistol_2.mp3, pistol_3.mp3
rifle_1.mp3, rifle_2.mp3, rifle_3.mp3
hit_1.mp3, hit_2.mp3, hit_3.mp3, hit_4.mp3
death_1.mp3, death_2.mp3, death_3.mp3
```

우선순위는 아래처럼 잡습니다.

```text
이벤트_상황_번호.mp3 > 이벤트_번호.mp3 > 이벤트.mp3 > 기존 합성 폴백
```

예시:

```text
death_elite_1.mp3
death_normal_1.mp3
explosion_large_1.mp3
hit_crit_flesh_1.mp3
```

## 7. BGM 파일

| 파일명 | 사용 상황 | 권장 길이 | 방향 |
|---|---|---:|---|
| `menu_loop.mp3` | 시작 화면, 상점 | 45~90 s 루프 | 어두운 학교 복도 분위기, 긴장감은 낮게 |
| `game_loop.mp3` | 인게임 전투 | 60~120 s 루프 | 빠른 반복 플레이에 피로하지 않은 전투 리듬 |

BGM은 효과음보다 훨씬 작게 둡니다. 기본 권장 믹스는 SFX 기준 BGM -12~-18 dB입니다.

## 8. 연동 메모

샘플을 실제 코드에 연결할 때는 쥬얼리아와 같은 순서가 좋습니다.

1. `SFX_ASSETS` 맵 추가: `start`, `skill`, `core`, `hit`, `crit`, `death`, `explosion`, `pistol`, `rifle`, `sniper`, `rocket`, `arrow`
2. 시작 시 `fetch` + `decodeAudioData`로 비차단 프리로드
3. `playSfx(name, intensity)`에서 샘플이 있으면 샘플 재생, 없으면 기존 합성음 실행
4. 반복 사운드는 최소 재생 간격(`minGap`) 유지
5. 샘플 추가 후 캐시/서비스워커를 쓰게 되면 캐시 목록과 버전도 같이 갱신

## 9. 우선 제작 순서

1. 무기 5종: `pistol`, `rifle`, `sniper`, `rocket`, `arrow`
2. 전투 반응 4종: `hit`, `crit`, `death`, `explosion`
3. UI/진행 3종: `start`, `skill`, `core`
4. BGM 2종: `menu_loop`, `game_loop`
5. 확장 UI/보상음: `button`, `purchase`, `denied`, `recruit`, `wave_clear`, `game_over`, `coin`, `pause`
